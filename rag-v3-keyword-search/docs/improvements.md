# Future Improvements

Version 3 implements a lexical retrieval engine based on keyword matching.

Although simple and efficient, lexical retrieval has several limitations.

## Current Limitations

* Cannot understand semantic meaning.
* Requires exact or near-exact keyword matches.
* Does not recognize synonyms.
* Cannot understand context.

---

## Planned Improvements

### Version 4

* Generate text embeddings
* Semantic document representation

### Version 5

* Store embeddings in a vector database

### Version 6

* Perform semantic retrieval using similarity search

### Version 7

* Integrate retrieval with a Large Language Model (LLM)

### Version 8

* Expose the system through FastAPI

### Version 9

* Build a web interface

## Long-Term Goal

Transform the project from a lexical retrieval system into a complete production-ready Enterprise Retrieval-Augmented Generation (RAG) platform capable of answering questions using private documents through semantic search and large language models.
