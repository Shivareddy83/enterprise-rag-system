# Contributing

Thank you for your interest in contributing to the Enterprise RAG System.

Contributions are welcome and appreciated.

---

# Development Setup

## 1. Clone the Repository

```bash
git clone https://github.com/shivareddy83/rag-v5-vector-database.git
```

---

## 2. Navigate to the Project

```bash
cd rag-v5-vector-database
```

---

## 3. Create a Virtual Environment

```bash
python -m venv .venv
```

---

## 4. Activate the Virtual Environment

### Windows

```bash
.venv\Scripts\Activate.ps1
```

### Linux / macOS

```bash
source .venv/bin/activate
```

---

## 5. Install Dependencies

```bash
pip install -r requirements.txt
```

---

## 6. Run the Application

```bash
python app.py
```

# Coding Guidelines

Please follow these guidelines when contributing:

- Write clean and readable code.
- Follow PEP 8 style guidelines.
- Use meaningful variable and function names.
- Add docstrings for all classes and methods.
- Keep modules focused on a single responsibility.
- Avoid duplicate code.

---

# Commit Messages

Use clear commit messages.

Examples:

```
Add ChromaDB integration

Fix duplicate embedding IDs

Improve terminal UI

Update documentation
```

---

# Pull Requests

Before submitting a pull request:

- Ensure the application runs successfully.
- Update documentation if necessary.
- Test your changes.
- Keep pull requests focused on a single feature or fix.

---

# Reporting Issues

When reporting an issue, include:

- Operating System
- Python Version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages (if any)

---

# Code of Conduct
